# VIBE_CODING — 猫猫助手团队产品规划

## 产品背景

这是主人 **minna** 的个人网站，主人的角色是在 **腾讯广告营销设计中心** 工作的 **UX 工程师**。

网站中有一群猫猫助手，每只猫有独立的职责和技能。正式上线后，每只猫都会接入 AI 大模型（Gemini / GPT 等），工作流也由 AI 驱动。当前阶段只考虑前端逻辑和 Mock 数据。

---

## 核心任务

| # | 任务 | 说明 |
|---|------|------|
| 1 | **自动爬取整理推送最新资讯** | 定时爬取指定网站/RSS，整理摘要，推送给主人 |
| 2 | **总结网站内容 + 自动学习 + 制定代办清单** | 分析网站现有内容，自主提出改进建议（发文章、增加页面、增加/调整猫猫等） |
| 3 | **根据主题和材料输出文章** | 主人提供主题和素材，猫猫生成完整文章 |
| 4 | **持续更新 Crafts 等创意页面** | 自动生成/更新前端组件、创意页面、交互 demo |
| 5 | **图片/视觉素材生成** | 为文章、页面生成配图和视觉素材 |
| 6 | **质量保障** | 内容审核、组件测试、质量检测 |
| 7 | **消息通知与分发** | 邮件推送、订阅管理、通知分发 |

---

## 猫猫团队（9只）

### 1. 小虎 — Manager（三花猫）
- **职责**：团队统筹调度、任务分配、流程审批、代办清单管理
- **核心能力**：
  - 分析网站当前内容，自动生成代办清单（发文章、增加页面、新增/调整猫猫职责等）
  - 将用户输入的任务拆解并分配给其他猫
  - 审批工作成果，决定是否发布
  - 协调多猫工作流的执行顺序
- **AI 接入**：LLM 做任务拆解 + 决策引擎

### 2. 雪 — Scout（黑猫）
- **职责**：资讯爬取、信息采集、数据分析
- **核心能力**：
  - 定时爬取指定网站/RSS/社交媒体，获取 UX/设计/前端 领域最新资讯
  - 对爬取内容进行摘要和分类
  - 网站数据分析（UV/PV/用户行为）
  - 生成趋势报告
- **AI 接入**：爬虫 + LLM 摘要 + 数据分析

### 3. 发发 — Writer（白猫）
- **职责**：文章撰写、内容润色、知识整理
- **核心能力**：
  - 根据主人提供的主题和材料生成完整文章
  - 将爬取的资讯整理为可发布的内容
  - 文本润色和风格调整
  - 生成文章大纲和结构化内容
- **AI 接入**：Gemini / GPT 文本生成

### 4. 花椒 — Builder（美短/起司猫）
- **职责**：前端组件生成、Crafts 页面持续更新
- **核心能力**：
  - 根据描述生成 React/HTML 创意组件
  - 持续更新 Crafts 页面（新增交互 demo、动画效果等）
  - 将文章+图片排版为精美页面
  - CSS/动画代码生成
- **AI 接入**：Gemini Code + 模板引擎

### 5. Pixel — Image Creator（暹罗猫）
- **职责**：图片生成、图表可视化、视觉素材
- **核心能力**：
  - AI 绘图（根据文字描述生成图片）
  - 数据可视化图表生成
  - 图片增强（超分辨率、降噪）
- **AI 接入**：Gemini Image / DALL-E

### 6. 年年 — Messenger（橘猫）
- **职责**：消息推送、邮件发送、订阅管理
- **核心能力**：
  - 发送 HTML 格式邮件
  - 向订阅者批量推送通知（新文章、新 Crafts 等）
  - 管理订阅者列表
  - 将爬取的资讯摘要推送给主人
- **AI 接入**：SMTP/SendGrid + WebPush

### 7. 墨墨 — Processor（橘白猫）
- **职责**：图片处理、格式转换、文字提取
- **核心能力**：
  - 图片像素化、滤镜处理
  - OCR 文字提取
  - 图片格式转换和批处理
- **AI 接入**：Canvas API + Tesseract + Sharp

### 8. 咪咪 — Audio Creator（白猫）
- **职责**：音乐生成、音效设计、音频处理
- **核心能力**：
  - 根据场景生成背景音乐
  - 音效合成
  - 多轨混音
- **AI 接入**：MusicGen / AudioCraft

### 9. 小白 — QA Inspector（奶牛猫）
- **职责**：质量检测、内容审核、自动化测试
- **核心能力**：
  - 对输出内容做质量评分
  - 内容合规审核
  - 页面组件回归测试
- **AI 接入**：Rules Engine + Puppeteer + Moderation API

## 协作工作流

### 📰 资讯采集与推送
`雪(爬取) → 发发(摘要整理) → 年年(推送给主人)`

### 📋 网站代办清单生成
`雪(分析网站内容) → 小虎(生成代办清单并分配任务)`

### ✍️ 文章发布
`发发(撰写文章) → Pixel(生成配图) → 花椒(排版成页面) → 小白(质量审核) → 年年(推送通知)`

### 🎨 Crafts 更新
`小虎(规划新 Craft) → 花椒(生成组件代码) → 小白(回归测试) → 年年(推送更新通知)`

### 📊 数据周报
`雪(采集分析数据) → Pixel(生成图表) → 发发(撰写报告) → 年年(邮件发送)`

### 🖼️ 图片处理流水线
`Pixel(生成原图) → 墨墨(处理/像素化) → 小白(质量检测)`

### 🎵 音乐视频制作
`咪咪(生成音乐) → Pixel(生成封面) → 花椒(制作播放页) → 年年(推送通知)`

### 🎯 创意企划
`花椒(构思方案) → Pixel(生成素材) → 发发(撰写文案) → 小虎(审批发布)`

## 技术栈

- **前端**: React + TypeScript + SCSS
- **后端**: Node.js + Prisma + PostgreSQL
- **AI**: Gemini API（文本/图片/代码生成）
- **部署**: Docker Compose
